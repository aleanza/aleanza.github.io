Scenario

The customer asks you to create a pilot for one of the company’s applications in public cloud. This is their first attempt to getting to know with non-on-prem environments

Tasks for this assignment

  1. Deliver the solution on a public cloud platform
  2. Present and highlight core concepts and benefits of the designed architecture.

Actual Architecture

  • Monolithic
  • Pros: Easy to develop and deploy
  • Cons: Not scalable, high coupling of modules, hard to develop by large team, hard to extend

Actual Solution

  • Available in a variety of web frameworks as a GIT repo
  • e.g. in Flask, a popular Python web application framework

Business Requirements

  1. Scalability: The Application must serve variable amount of traffic
  2. Recoverability: Preserve the notes up to 3 years, being able to recover them it if needed
  3. Resiliency: in case of datacenter failures, recovery to any regions supported by the cloud provider
  4. Resiliency: Concurrent development / deployment by a team of > 100 developers
  5. Support of development process by use of separated environments (development, testing, production)
  6. Monitoring of relevant metrics and logs from the infrastructure for quality assurance and security purposes.
